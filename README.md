# RAMC: Risk-Aware Model-then-Control for Predictive Thermal Control of Buildings

Code and synthetic data accompanying the article "Risk-Aware Model Training for Predictive Thermal Control of Buildings", submitted to *Buildings* (MDPI).

RAMC augments the conventional prediction loss of a learned thermal-dynamics model with a Conditional Value-at-Risk (CVaR) penalty on operational cost under perturbed inputs. This embeds tail-risk awareness directly into the prediction model that a nonlinear model predictive controller (NMPC) relies on, so the controller is hedged against worst-case operating conditions rather than only the average case.

## Pipeline overview

The repository is organized as an eight-stage pipeline; each numbered folder consumes the output of the previous one. A high-fidelity EnergyPlus model is used once, in Stage 1, to generate ground-truth data. A reduced-order resistance-capacitance (RC) model is then identified from that data and used as the plant for everything downstream. Because of this, only Stage 1 requires EnergyPlus; Stages 2 to 8 run on the identified RC model alone.

| Stage | Folder | Purpose |
| ----- | ------ | ------- |
| 1 | `1_Building Simulation` | EnergyPlus model of an ASHRAE 90.1 Retail Standalone prototype building adapted to Västerås (gas furnace replaced by a hydronic district-heating loop). Produces the raw simulation output and the energy and thermal-comfort analyses. |
| 2 | `2_Dataset Creation for RC model` | Cleans and reshapes the EnergyPlus output into the processed dataset used for system identification (`DataPreprocessing.py`). |
| 3 | `3_Master Code for System Identification` | Identifies the gray-box RC parameters by differential evolution for radiator discretizations Nr = 1, 2, 3 (`SystemIdentification.py`); results in `results_N1/N2/N3_DE_optimized.json`. |
| 4 | `4_Universal Simulator Validation for RC` | Validates the identified RC model on held-out data (April) and performs residual analysis (`validate_rc_on_april.py`, `ResidualAnalysis.py`). |
| 5 | `5_Data Generation for Neural Network` | Generates the RAMC training set from the RC plant under perturbed inputs (`Training_Data_Generation.py`). The large raw training CSV is regenerable and is not tracked; its metadata is in `RAMC_training_data_N3_meta.json`. |
| 6 | `6_Neural Network Architecture` | Trains the thermal-dynamics network with the RAMC (CVaR) loss and runs the main experiments and ablations (`main_experiment.py`, `trainers.py`, `ramc_losses.py`, `thermal_dynamics_net.py`). |
| 7 | `7_Close Loop Performance` | Closed-loop evaluation: weather scenarios (`M1_Weather_Data`), the RC-based closed-loop simulator (`M4_Closed_Loop_Simulator`), and the direct-shooting NMPC (`M6_NMPC_Direct_Shooting`). |
| 8 | `8_Assignments` | Supporting studies referenced in the paper: external validity (A1), benchmark sufficiency (A2), lambda selection (A3), controller robustness (A4), mechanistic bridge (A5), Pareto analysis (A6), system-identification transparency (A7), and zone-aggregation audit (A8). Shared utilities are in `8_Assignments/shared`. |

## Requirements

- Python 3.9 or newer.
- Python dependencies, listed in `requirements.txt`:

  ```bash
  pip install -r requirements.txt
  ```

- EnergyPlus v25.1 (https://energyplus.net), required only to regenerate Stage 1 data. Stages 2 to 8 do not need it.

The weather file (`Vasteras2007_2021.epw`) and the processed input data are included in the repository, so the RC-model and control results can be reproduced without re-running EnergyPlus.

## Reproducing the results

Run the stages in order. The main script in each stage is listed below; see the top of each script for paths and options.

1. `1_Building Simulation/AnalyzeDistrictHeating.py` (requires EnergyPlus)
2. `2_Dataset Creation for RC model/DataPreprocessing.py`
3. `3_Master Code for System Identification/SystemIdentification.py`
4. `4_Universal Simulator Validation for RC/validate_rc_on_april.py`
5. `5_Data Generation for Neural Network/Training_Data_Generation.py`
6. `6_Neural Network Architecture/main_experiment.py`
7. `7_Close Loop Performance/M6_NMPC_Direct_Shooting/run_nmpc_matrix.py`
8. `8_Assignments/A1_external_validity/run_a1.py` through `8_Assignments/A8_zone_aggregation_audit/run_a8.py`

Figures are regenerated by the plotting scripts in Stages 6 and 7, such as `make_fig6_rollout_2panel.py` and `plot_nmpc_results_final.py`. Generated figures, model weights, and large output folders are excluded from version control and are reproduced by re-running the scripts above.

## Data availability

The synthetic data needed to reproduce the RC model and the closed-loop results are included in this repository. The large neural-network training CSV is regenerable from Stage 5 and is therefore not tracked.
